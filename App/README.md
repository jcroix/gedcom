# GedReader (macOS app)

The SwiftUI app on top of the `GedcomKit` engine + `GedReaderCore` logic library (both in the SPM
package one directory up). The app target is a **thin view layer**; all testable logic lives in
`GedReaderCore` and is covered by `swift test` at the repo root.

## Project file is generated

`GedReader.xcodeproj` is **generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)** from
`project.yml` — that YAML is the source of truth, not the `.xcodeproj` (which is gitignored). After
changing `project.yml` (or after a fresh clone), regenerate:

```bash
cd App
xcodegen generate          # requires: brew install xcodegen
```

## Build & run

The easiest way is the **Makefile at the repo root** (it handles xcodegen + xcodebuild for you):

```bash
make run        # build the app and launch it
make app        # build only (Debug);   make release  for Release
make test       # run all unit tests
make help       # list all targets
```

Equivalent raw commands, if you prefer:

```bash
cd App
xcodegen generate
xcodebuild -project GedReader.xcodeproj -scheme GedReader \
  -configuration Debug -destination 'platform=macOS' -derivedDataPath build build
open build/Build/Products/Debug/GedReader.app
```

Note: no Apple Developer signing identity is configured, so builds are **ad-hoc signed**
(`CODE_SIGN_IDENTITY = -`) — fine for local running, not for distribution.
